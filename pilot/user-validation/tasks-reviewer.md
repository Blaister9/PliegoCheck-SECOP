# Tareas REVIEWER

Datos: usar `reviewer@pilot.pliegocheck.local`.

| paso | tarea                              | resultado esperado                                   |
| ---- | ---------------------------------- | ---------------------------------------------------- |
| 1    | Login                              | Acceso exitoso con permisos de revision.             |
| 2    | Revisar resultados                 | Decision, hallazgos y acciones visibles.             |
| 3    | Confirmar resultado                | Confirmacion auditada sin recalcular motor.          |
| 4    | Hacer override sintetico con razon | Override permitido si se justifica; queda auditado.  |
| 5    | Revisar decision efectiva          | `effective_outcome` refleja confirmacion u override. |
| 6    | Revisar acciones                   | Acciones siguen trazables a hallazgos.               |
| 7    | Descargar reporte si aplica        | Descarga permitida.                                  |
| 8    | Logout                             | Sesion cerrada.                                      |

No emitir concepto juridico real. El objetivo es validar la mecanica de revision.
